# Origin Configuration Loader

A command-line tool for managing origin configurations in the storage database.

## Features

- **Load configurations** from a text file with hostname and JSON config per line
- **Upsert functionality** - automatically creates or updates existing configurations
- **Delete by hostname** - remove specific configuration
- **Delete by prefix** - remove multiple configurations matching a prefix (with safety checks)
- **UUID generation** - automatically generates UUIDs for new entries
- **Multi-database support** - works with PostgreSQL and SQLite via DSN

## Installation

```bash
cd /Users/rick/projects/proxy/tools/config-loader
go build -o config-loader
```

## Usage

### Load Configurations from File

```bash
# Using SQLite
./config-loader -dsn sqlite:///tmp/config.db -load configs.txt

# Using PostgreSQL
./config-loader -dsn 'postgres://user:pass@localhost:5432/db?sslmode=disable' -load configs.txt

# Using environment variable for DSN
export STORAGE_DSN='sqlite:///tmp/config.db'
./config-loader -load configs.txt
```

### Delete Specific Configuration

```bash
./config-loader -dsn sqlite:///tmp/config.db -delete api.soapbucket.com
```

### Delete by Prefix

```bash
# Will prompt for confirmation
./config-loader -dsn sqlite:///tmp/config.db -delete-prefix 'api.'
```

**Safety Features:**
- Cannot delete with empty prefix (prevents accidental deletion of all records)
- Minimum prefix length of 2 characters
- Interactive confirmation prompt

### List Configurations

```bash
./config-loader -dsn sqlite:///tmp/config.db -list
```

Note: Listing shows available database commands. For actual listing, use database tools directly.

## Configuration File Format

Each line should contain:
```
hostname json-config
```

The JSON config should follow the Origin structure from the proxy:

```json
{
  "hostname": "api.example.com",
  "type": "proxy",
  "config": {
    "url": "https://backend.example.com",
    "timeout": "30s",
    "maxConnections": 100,
    "rateLimit": 1000
  }
}
```

**Note:** The `id` field is automatically generated as a UUID and injected into the origin config when loaded.

Example `configs.txt`:
```
# Comments start with #
api.example.com {"hostname":"api.example.com","type":"proxy","config":{"url":"https://backend.example.com","timeout":"30s"}}
www.example.com {"hostname":"www.example.com","type":"proxy","config":{"url":"https://cdn.example.com","timeout":"20s"}}
localhost:8080 {"hostname":"localhost:8080","type":"proxy","config":{"url":"https://httpbin.org","timeout":"45s"}}
```

### Example Configuration

See `configs.example.txt` for complete examples with:
- **api.soapbucket.com** - Proxy to Hacker News with full transport settings
- **api.soapbucket.com:8443** - Forward hostname configuration
- **localhost:8443** - Local development proxy to example.com

See `configs.example.pretty.json` for a formatted view of the configurations.

## DSN Formats

### SQLite
```
sqlite:///path/to/database.db

```

### PostgreSQL
```
postgres://user:password@host:port/database?sslmode=disable
postgres://user:password@host:port/database?sslmode=require
```

## How It Works

### Load Operation
1. Reads file line by line
2. Parses hostname and JSON configuration
3. Generates a UUID and injects it as the `id` field in the origin config
4. Calls `storage.Put()` which performs an upsert:
   - If hostname doesn't exist: creates new entry with auto-generated storage UUID
   - If hostname exists: updates the value (including new origin id), preserves storage UUID
5. Reports success/failure for each line with the origin id

**Two UUIDs:**
- **Storage UUID** - Primary key in the database (auto-generated by storage layer)
- **Origin ID** - The `id` field in the origin JSON config (auto-generated by config-loader)

### Delete Operations
- **Delete by hostname**: Removes exact match using `storage.Delete()`
- **Delete by prefix**: Removes all matching hostnames using `storage.DeleteByPrefix()`
  - Safety checks prevent accidental deletion
  - Requires confirmation prompt

### Database Schema

The tool uses the storage layer which creates this schema:

**PostgreSQL:**
```sql
CREATE TABLE config_storage (
    id UUID PRIMARY KEY,
    key VARCHAR(255) NOT NULL,
    value BYTEA NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_config_storage_key ON config_storage(key);
```

**SQLite:**
```sql
CREATE TABLE config_storage (
    id TEXT PRIMARY KEY,
    key TEXT NOT NULL,
    value BLOB NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_config_storage_key ON config_storage(key);
```

## Examples

### Quick Start with SQLite

```bash
# Build the tool
cd /Users/rick/projects/proxy/tools/config-loader
go build -o config-loader

# Load example configurations
./config-loader -dsn sqlite:///tmp/origins.db -load configs.example.txt

# Check what was loaded (using sqlite3)
sqlite3 /tmp/origins.db "SELECT key FROM config_storage"

# Delete a specific config
./config-loader -dsn sqlite:///tmp/origins.db -delete localhost:8443

# Delete all api.soapbucket.com configs
./config-loader -dsn sqlite:///tmp/origins.db -delete-prefix 'api.soapbucket.com'
```

### Quick Start with PostgreSQL + Docker

```bash
# Start PostgreSQL
docker run -d --name config-db \
  -e POSTGRES_PASSWORD=secret \
  -p 5432:5432 \
  postgres:16-alpine

# Wait for it to be ready
sleep 3

# Load configurations
./config-loader \
  -dsn 'postgres://postgres:secret@localhost:5432/postgres?sslmode=disable' \
  -load configs.example.txt

# Query using psql
docker exec -it config-db psql -U postgres -c "SELECT key, value FROM config_storage"

# Cleanup
docker stop config-db && docker rm config-db
```

## Error Handling

The tool provides detailed error messages for:
- Invalid DSN format
- Database connection failures
- File not found or read errors
- Invalid configuration format
- Database operation failures

Exit codes:
- `0` - Success
- `1` - Error occurred

## Safety Features

1. **Empty prefix protection** - Cannot delete with empty prefix
2. **Minimum prefix length** - Prefix must be at least 2 characters
3. **Confirmation prompt** - Interactive confirmation for prefix deletion
4. **Comment support** - Lines starting with `#` are ignored
5. **Line validation** - Checks for proper format before storing
6. **Context timeout** - 30-second timeout for all operations

## Integration

## Troubleshooting

### "DSN is required"
Set the DSN via `-dsn` flag or `STORAGE_DSN` environment variable.

### "Error creating storage"
Check DSN format and database connectivity. For PostgreSQL, ensure the server is running and accessible.

### "Invalid format"
Each line must have format: `hostname json-config` with a space separator.

### "Permission denied"
Ensure database file (SQLite) is writable or database user (PostgreSQL) has proper permissions.
