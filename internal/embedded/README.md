# Embedded Data Package

This package embeds pre-compressed data files into the proxy binary at compile time using Go's `//go:embed` directive. This eliminates external file dependencies at runtime.

## Files

| File | Description | Source |
|------|-------------|--------|
| `ai_providers.yml.gz` | AI provider definitions (endpoints, auth patterns, model mappings) | Internal provider registry |
| `regexes.yml.gz` | User-agent parsing rules for device and browser detection | [ua-parser/uap-core](https://github.com/ua-parser/uap-core) (Apache 2.0) |
| `version.json` | Metadata (SHA256 hashes, sizes, timestamps) for each embedded file | Generated by update script |

All `.gz` files are gzip-compressed with maximum compression (`gzip -9`). They are decompressed in memory on first use.

## Upstream Sources

- **ai_providers.yml** - Internal sbproxy provider registry. Maintained in this repository.
- **regexes.yml** - From the [ua-parser/uap-core](https://github.com/ua-parser/uap-core) project (Apache 2.0 license). Contains regular expressions for parsing User-Agent strings into browser, OS, and device information.

## Overriding Embedded Files

Users can override any embedded file by providing a path in the proxy configuration. The embedded file is used as a fallback when no override is configured.

When an override path is set, the proxy logs which file is used (embedded vs external) along with its SHA256 hash.

## Updating Files

To regenerate the embedded copies from their upstream sources, run:

```bash
./scripts/update-embedded-data.sh
```

The script downloads or copies each source file, compresses it with `gzip -9`, computes SHA256 checksums, and writes `version.json`.
