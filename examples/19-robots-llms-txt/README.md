# 19 - robots.txt + llms.txt projections
*Last modified: 2026-05-02*

Demonstrates the Wave 4 text-format policy-graph projections.
Cooperative crawlers fetch one of three documents to discover an
origin's pricing posture without hitting the priced content paths:

| Path             | Format                        | Audience                   |
|------------------|-------------------------------|----------------------------|
| `/robots.txt`    | RFC 9309 + RSL stanzas        | Standard web crawlers.     |
| `/llms.txt`      | YAML-ish header + summary     | LLM training crawlers.     |
| `/llms-full.txt` | Header + priced-route listing | LLM training crawlers.     |

All three are derived from the `ai_crawl_control` policy plus the
per-origin `content_signal` field. Editing the policy regenerates the
projections atomically on the next config reload.

## How it composes

| Service     | Image                              | Role                                                          |
|-------------|------------------------------------|---------------------------------------------------------------|
| sbproxy     | built from `Dockerfile.cloudbuild` | Proxy + projection cache. Serves the three text projections.  |

The proxy is the only container; there is no upstream service to
healthcheck.

## How to run

```bash
cd examples/19-robots-llms-txt
docker compose up -d --wait
```

Tear down:

```bash
docker compose down -v
```

The `Makefile` wraps the same calls (`make up`, `make down`,
`make logs`, `make test`).

## What to expect

### 1. `/robots.txt`

Per-agent stanzas in RFC 9309 format. The Wave 4 projection adds the
non-standard `# SBproxy-AI-Extension:` comment carrying pricing
metadata for cooperative crawlers; standard crawlers skip the comment
and obey the `Disallow` rules.

```bash
curl -s -H 'Host: news.localhost' \
     http://127.0.0.1:8080/robots.txt
# # Generated by SBproxy. Do not edit.
# # Config version: 42
#
# User-agent: GPTBot
# Disallow: /feed/*
# Disallow: /article/*
# # SBproxy-AI-Extension: pay-per-crawl price=0.005 currency=USD shape=markdown
# # SBproxy-AI-Extension: pay-per-crawl price=0.001 currency=USD shape=html
#
# ...
```

### 2. `/llms.txt`

Concise summary. The header carries the sitename, config version, the
operator's licensing posture, and the offered shapes; the body is a
one-paragraph summary that points the crawler at `/llms-full.txt` for
the per-route listing.

```bash
curl -s -H 'Host: news.localhost' \
     http://127.0.0.1:8080/llms.txt
# # sitename: news.localhost
# # version: 42
# # payment: pay-per-request
# # shapes: html, markdown
# # license: ai-train
#
# # news.localhost
#
# Crawler pricing for `news.localhost` is enforced by SBproxy.
# Cooperative agents should fetch `/llms-full.txt` for the full
# priced-route listing, or `/.well-known/tdmrep.json` for the
# machine-readable text-and-data-mining policy.
```

### 3. `/llms-full.txt`

Same header plus a `## Priced routes` section listing every
`(route_pattern, content_shape, price)` tuple from the
`ai_crawl_control` config.

```bash
curl -s -H 'Host: news.localhost' \
     http://127.0.0.1:8080/llms-full.txt
# # sitename: news.localhost
# # version: 42
# ...
#
# ## Priced routes
#
# - `/feed/*` (markdown): 0.005000 USD per request
# - `/feed/*` (html):     0.001000 USD per request
# - `/article/*` (html):  0.001000 USD per request
```

## How agent classes map

The Wave 4 `ai_crawl_control` policy keys tiers on `(route_pattern,
content_shape)`. Per-agent pricing surfaces in the projection because
the cache renders one `robots.txt` stanza per agent class declared in
`crawler_user_agents:`. An agent that does not appear there is treated
as a regular browser and bypasses pricing.

| Crawler                   | Default tier (per `/article/*`) |
|---------------------------|---------------------------------|
| GPTBot                    | $0.001 / request                |
| ChatGPT-User              | $0.001 / request                |
| ClaudeBot                 | $0.001 / request                |
| anthropic-ai              | $0.001 / request                |
| PerplexityBot             | $0.001 / request                |
| Google-Extended           | $0.001 / request                |
| CCBot                     | $0.001 / request                |
| (anything else)           | bypass (no charge)              |

The `/feed/*` path lifts agents that send `Accept: text/markdown` to
the $0.005 markdown tier; the html shape stays at $0.001.

## Operator workflow

1. Edit `tiers:` in `sb.yml` to add or remove a route.
2. Reload the proxy (`docker compose restart sbproxy`).
3. The projection cache rebuilds atomically. Each refresh emits one
   `PolicyProjectionRefresh` audit event per (hostname,
   projection_kind) so the operator can trace any change back to the
   originating edit.
